Why Safwa AI

Built for banks, not browsers

Every feature was designed around Safwa Bank's strict data sovereignty, compliance, and usability requirements.

100% Local Processing
All AI inference runs on-device via Ollama. No prompts, no documents, no answers ever touch an external server or cloud API.
28 Arabic Policy Documents
Ingests all official regulations from Jordan's Ministry of Digital Economy — automatically chunked, embedded, and indexed for instant retrieval.
Source Citations Every Time
Every answer includes the exact document name, page estimate, and section — so employees can verify directly from the source.
Role-Aware Responses
Adapts answer depth and terminology based on the employee's role: IT, Business, Management, HR, Legal, or General Staff.
Bilingual (AR + EN)
Ask in Arabic and receive an Arabic answer. Ask in English and receive an English answer. Automatically detected — no language setting needed.
No Hallucinations
Strictly document-grounded. The system refuses to answer questions that fall outside the ingested policy corpus — no guessing allowed.
Full Conversation History
All chat sessions stored locally in SQLite. Employees can review past queries and answers at any time without re-asking.
Admin Ingestion Panel
Re-ingest documents anytime via a hidden admin panel (Ctrl+Shift+A). Live progress indicator and LLM health check included.

Quick Start

Up and running in 5 steps

First-time setup takes ~15 minutes (mostly download time). After that, launches instantly.

1
Install Ollama (Local LLM Server)
Download from ollama.com or use winget:
winget install Ollama.Ollama
2
Pull the Mistral 7B Model (~4.4 GB, one-time)
ollama pull mistral
3
Clone the repository & install dependencies
git clone https://github.com/RaneemSadeh/safwa-ai-assistant.git
cd safwa-ai-assistant
pip install -r requirements.txt
4
Add your `.docx` policy files to Data/ then ingest
python src/ingest.py
5
Start the application
python app.py
Open your browser at http://localhost:5000
